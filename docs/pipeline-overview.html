8
SQLValidator Hybrid gpt-4o-mini (opsional)

Memvalidasi SQL yang dihasilkan sebelum dieksekusi ke database. Tiga lapis deterministik + satu lapis LLM opsional.

  • 1
    Syntax — parsing via sqlparse. Mendeteksi SQL tidak valid secara sintaks.
  • 2
    Security — blocklist 12 keyword berbahaya:
    DROPDELETE TRUNCATEALTER CREATEINSERT UPDATEGRANT REVOKEEXECUTE PROCEDUREFUNCTION
  • 3
    Whitelist tabel — hanya 10 tabel yang diizinkan:
    daily_master financial_internal product_summary channel_payment product_price_list anomalies daily_unique_users daily_user_partner hourly_pattern_daily daily_product_channel
  • 4
    LLM Logic check (opsional, ENABLE_AI_VALIDATION=true) — verifikasi apakah SQL masuk akal secara bisnis. Menggunakan model lebih kecil (gpt-4o-mini).
Output state.validated_sql
9
QueryExecutor Traditional Tanpa LLM

Menjalankan validated SQL ke PostgreSQL menggunakan SQLAlchemy connection pool. Jika gagal, melempar QueryExecutionError yang ditangkap pipeline untuk retry ke SQLGenerator.

Timeout 30 detik per query Maks baris 10.000 baris Pool size 5 koneksi aktif Max overflow 10 koneksi tambahan Recycle 1800 detik (30 menit) Database PostgreSQL 14+ · financial_db Output state.query_result[], state.row_count
🔁 Jika QueryExecutor gagal → pesan error dikembalikan ke SQLGenerator · maks 2× retry
